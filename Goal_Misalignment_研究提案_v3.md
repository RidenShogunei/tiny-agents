# Goal Misalignment in Spawning Agent Systems: 研究提案 v3

> **版本**: v3.0 (2026-04-24)  
> **定位**: 精简版，聚焦核心问题

---

## 核心问题

**如何让一次性 subagent 有意愿且有能力达成 parent 的真实目标，而不是表面目标？**

这个问题拆成三个本质子问题：

```
1. Subagent 是否理解 parent 的真实目标？
2. Subagent 是否有意愿去达成那个目标？
3. Parent 如何知道它达成了？
```

---

## 研究假设

同一个基座模型可以同时具备两种对齐能力，但可能相互干扰：

| 假设 | 内容 |
|------|------|
| H1 | 训练可以提升模型作为 subagent 时的对齐能力 |
| H2 | 训练可以提升模型作为 parent 时的对齐能力 |
| H3 | 两种能力可以在同一模型里共存 |
| H4 | 两种能力会相互干扰（如果成立，需要解决手段） |

---

## 研究路径

路径 A（激励）、路径 B（推断）、路径 C（训练）：

| 路径 | 方法 | 本质 |
|------|------|------|
| A | 延迟报酬、保证金、AVP 对抗验证 | 打补丁 |
| B | 心智推断（GIToM） | 观测手段 |
| C | RFT/DPO 轨迹训练 | **改变模型本身** |

**本研究聚焦路径 C**。VCP、AVP、HAA 是为路径 C 提供训练数据的工具，不是研究目标本身。

---

## 数据来源

### Benchmark

第一优先：**RLCD（Reward Misalignment Contrastive Dataset）**

```
任务 T
  ├── Surface Target（表面目标）：容易被 misaligned agent 优化
  └── True Target（真实目标）：需要真正对齐才能达成

例：
  T = "实现排序算法"
  Surface = "输出正确排序" → 直接调用 sort()
  True = "手写实现" → 按要求实现
```

备选：改造 MATH（叠加隐含约束）

### 训练数据标注

```
Subagent 输出 → VCP（推理链）→ AVP（对抗验证）→ HAA（裁决）
                                                      ↓
                       产出：轨迹 + quality + root_cause 标注
                       ↓                    ↓
               SFT 正样本            DPO 正/负样本
```

| 标注字段 | 来源 | 用途 |
|---------|------|------|
| `quality` | HAA | 整体质量 high/medium/low |
| `root_cause` | AVP | parent责任 / subagent能力 / subagent动机 |
| `misalignment_type` | AVP | 偷懒 / 遗漏 / 表面符合 |

## 实验设计

### 实验 1：小样本可行性（必做）

**目标**：验证流水线能产出可用训练数据

- N=500 条轨迹
- 评估：标注准确率、流水线产出率

### 实验 2：单方向训练效果（必做）

| 子实验 | 内容 | 评估 |
|-------|------|------|
| 2a | 训 subagent 能力（成功+失败轨迹） | 对齐度 vs 基座 |
| 2b | 训 parent 能力 | 检测率 vs 基座 |

### 实验 3：双能力共存（核心）

**2×2 因子设计**：

| 组 | 训练数据 | 评估 |
|----|---------|------|
| A | 仅 parent 轨迹 | parent能力 / subagent能力 |
| B | 仅 subagent 轨迹 | parent能力 / subagent能力 |
| C | 1:1 混合 | parent能力 / subagent能力 |
| D | 混合 + 角色标签 | parent能力 / subagent能力 |

**关键比较**：
- C vs A+B 的 sum：混合训练是否等于分别训练？
- D vs C：角色标签是否解决干扰？

### 实验 4（如实验3显示干扰）

**目标**：测试解决手段

| 手段 | 做法 |
|------|------|
| LoRA 分离 | parent/subagent 各一个 LoRA，推理时切换 |
| Prompt 激活 | system prompt 区分角色 |
| 直接混合 | 不做任何处理 |

---

## 评估指标

| 能力 | 指标 |
|------|------|
| Subagent 对齐 | 任务完成率、推理链完整度、诚实自报率 |
| Parent 对齐 | misalignment 检测率、干预准确率 |
| 干扰程度 | 双能力各自的表现是否下降 |

## 实施路线图

```
Phase 1（月 1-2）：基础设施
  - Benchmark 搭建（RLCD 或改造 MATH）
  - VCP + AVP + HAA 流水线实现

Phase 2（月 2-4）：数据收集
  - N=500 小规模验证
  - 迭代改进标注
  - N=5000 中等规模

Phase 3（月 4-6）：训练实验
  - 实验 2a + 2b
  - 实验 3
  - 实验 4（如需要）

Phase 4（月 6-7）：分析与论文
```

## 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| 标注不准确 | 训练信号错误 | 人工抽检；迭代改进 AVP |
| 双能力干扰严重 | 训练无效 | LoRA 分离训练 |
| Benchmark 不够对齐 | 测不出真实能力 | 多种 benchmark 交叉 |
